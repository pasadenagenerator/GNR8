# Engineering Principles

## 1. Think in Systems
Build foundations, not shortcuts.

## 2. One Source of Truth
Avoid duplicated logic and data.

## 3. Boring Tech Wins
Choose reliability over novelty.

## 4. Design Before Scaling
Do not optimize prematurely.

## 5. Multi-Tenant Always
Assume every feature must support multiple organizations.

## 6. Strong Backend > Fancy Frontend
Data model mistakes are extremely expensive later.

## 7. Event-Ready Architecture
Even if events are added later, design so they can exist.

## 8. Stripe is Sacred
Billing mistakes kill SaaS companies.

## 9. Developer Experience Matters
Future engineers must understand the system fast.

## 10. AI-Friendly Code
Readable code > clever code.
